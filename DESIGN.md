# Adapted Marketplace Design Notes

## North Star

The marketplace should feel like an obsidian control surface for serious creators:

- premium, dark, deliberate
- layered by tone, not by borders
- editorial, not template-like
- technical, but still intuitive on first glance

## Practical Rules

- Prefer less visible text on product pages.
- Keep the primary action obvious.
- Move secondary explanations behind contextual info affordances.
- Use delayed hover tooltips for quick clarification.
- Link those info affordances to focused help pages when deeper guidance exists.
- Use animated tile art to show that these plugins are live control surfaces, not static brochures.

## Layout

- No hard divider-line design language.
- Use tonal separation and layered surfaces instead.
- Keep navigation obvious and calm.
- Let the hero and plugin visuals carry part of the explanation.
- Sidebars should stay concise. Dense technical detail belongs in tooltips or focused help routes.

## Motion

- Animation should feel precise and restrained.
- Prefer slow drift, soft glow, and subtle hover lift.
- Avoid gimmicky motion.

## Typography

- Headlines should be assertive and compact.
- Body copy should be short and scannable.
- Metadata should be tight and legible.

## Marketplace UX

- Marketplace page: fast scan, fast filter, strong featured card.
- Plugin page: clear value, real visuals, obvious install path, focused help.
- Help page: one topic per page, not a generic documentation wall.
